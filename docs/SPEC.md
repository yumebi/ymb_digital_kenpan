# DigitalKenpan.jsx 仕様詳細

チェック項目の判定ロジック・制限事項・開発者向け情報をまとめたドキュメントです。
インストール方法と基本的な使い方は [README](../README.md) を参照してください。

## 1. 事前設定項目

| 設定 | 既定値 | 説明 |
|---|---|---|
| 仕上がりサイズ | A4 | A3/A4/A5/B4/B5/B6/ハガキ(100×148)/名刺(91×55)/カスタム。縦横は自動判定(入替許容) |
| 許容誤差 | ±0.5mm | サイズ照合の許容誤差 |
| 印刷カラー数 | 4色CMYK | 1色モノクロ/4色CMYK/4色+特色/特色のみ。カラー系チェックの判定基準が連動 |
| 想定特色数 | 0 | 「4色+特色」「特色のみ」時に使用特色数と照合(0=照合しない) |
| 塗り足し幅 | 3mm | 断ち落とし要素に必要な塗り足し量 |
| 文字セーフマージン | 5mm | 仕上がり枠から内側に必要な文字の余白 |
| 最小線幅 | 0.25pt | これ以下の線を極細線としてエラー |
| 画像解像度(カラー/グレー) | 下限300dpi / 過剰900dpi超 | 実効解像度の判定閾値 |
| 画像解像度(モノクロ2値) | 下限600dpi / 過剰1200dpi超 | 1bit画像(ビットマップ)用の判定閾値 |
| インキ総量上限 | 300% | プリセット(標準300%/油性350%/UV380%)から選択も可 |
| ラスタライズ効果解像度下限 | 300ppi | ドキュメント設定の解像度チェック |
| オブジェクト総数警告閾値 | 100,000件 | RIP負荷警告 |
| 細ケイ+薄アミ閾値 | 0.5pt以下かつ濃度20%以下 | 飛び・カスレ警告 |
| 各チェックのON/OFF | 全ON | チェックボックスで個別に無効化可能 |

### 設定ファイル

「設定を保存」ボタンでスクリプトと同じフォルダに `DigitalKenpan_settings.txt`(key=value 形式)が作成され、次回起動時に自動で読み込まれます。手動編集も可能です。

## 2. チェック項目と判定ロジック

判定は4値です。**判定不能な場合は誤OKを出さず、必ず「警告(要確認)」に倒します。**

- **OK** — 問題なし
- **エラー(NG)** — 入稿前に修正が必要
- **警告(要確認)** — 自動判定できない/意図的な場合がある。目視確認が必要
- **情報** — 列挙のみ(問題とは限らないが把握しておくべき内容)

### サイズとトンボ

| 項目 | ロジック | 判定 |
|---|---|---|
| データサイズ照合 | アートボードを mm 換算し仕上がりサイズと照合(縦横入替許容)。不一致時は (1)仕上がりサイズ相当の矩形パス (2)アートボード−塗り足し の順で照合を試行 | 一致=OK / 推定一致・不明=警告(検出サイズ表示) |
| 塗り足し | 仕上がり枠をまたぐオブジェクトのうち、はみ出し側の端が仕上がり枠+塗り足し幅に届かないものを辺ごとに列挙 | 不足=エラー |
| トンボ有無 | 仕上がり枠外〜15mm範囲の細線(2点・開いたパス・1.2pt以下)を計数し、4本以上でトンボありと推定 | あり=OK / なし=警告(PDF書き出し時付与の運用があるためNGにしない) |

### フォント

| 項目 | ロジック | 判定 |
|---|---|---|
| アウトライン化 | TextFrame が存在するか。存在する場合は先頭20文字・フォント名・位置を列挙 | 存在=エラー。シンボル・グラフ内テキストは検出不可の場合あり(注記) |
| 文字セーフマージン | 仕上がり枠から内側マージンラインを越える TextFrame を列挙 | 越え=エラー / テキスト無し=スキップ(−表示) |

### カラー

| 項目 | ロジック | 判定 |
|---|---|---|
| RGB混入 | documentColorSpace が RGB なら即エラー。各オブジェクトの塗り/線/グラデーションストップの RGBColor を列挙 | 検出=エラー |
| 特色 | SpotColor 使用を列挙し印刷カラー数設定と照合。4色CMYK設定で特色=エラー、1色モノクロでK以外=エラー、4色+特色/特色のみは特色名一覧+想定数照合。レジストレーションカラーのトンボ以外での使用=エラー | 設定連動 |
| インキ総量 | CMYK塗り/線/グラデーションストップの C+M+Y+K が上限超のものを色値付きで列挙。特色は代替色定義×濃度からの近似 | 超過=エラー。画像内部は対象外(注記) |
| リッチブラック/4Cブラック | K70%以上かつ C/M/Y いずれか5%超。テキスト・2pt以下の線・約1000mm²未満の塗りを「小サイズ」と判定 | 小サイズ=エラー(見当ズレ事故) / 大面積=情報 |
| グレースケールカラー使用 | GrayColor の塗り/線を列挙 | 警告。1色モノクロ設定時は情報(OK扱い) |
| 特色の品質 | 特色名の禁止/機種依存文字(& < > " ' / \ % # ; :)、代替色定義の色空間(Lab/CMYK)表示、同名(名前ゆれ)で定義が異なる特色 | 禁止文字・重複定義=警告 / 定義表示=情報 |
| アートボードごとの使用色数 | 各アートボード上のベクトルオブジェクトから版(C/M/Y/K/特色)を集計し 1C〜5C以上を表示。印刷カラー数設定と照合 | 不一致=エラー / 画像あり=警告(画像内の色は未集計と注記) / 整合=情報 |
| 白ページ(空アートボード) | 表示状態の描画オブジェクトが1つも重ならないアートボード | 警告 |

### 画像

| 項目 | ロジック | 判定 |
|---|---|---|
| リンク切れ | placedItem.file の存在確認(参照エラー含む) | 切れ=エラー |
| 画像カラーモード | 埋め込み(RasterItem)は imageColorSpace。リンク(PlacedItem)はファイルヘッダを自前解析(JPEG SOF成分数 / PNG / TIFF / PSD / BMP / GIF) | RGB=エラー / 判定不能形式=警告 |
| 実効解像度 | リンク画像はヘッダのピクセル数と配置サイズ(matrix の拡大率考慮)から実効dpi算出。カラー/グレーとモノクロ2値で閾値を分離。過剰解像度も警告 | 下限未満=エラー / 過剰=警告 / 埋め込み画像・ヘッダ不明=警告(判定不能) |

### オブジェクト・効果

| 項目 | ロジック | 判定 |
|---|---|---|
| 透明効果 | blendingMode が通常以外、または opacity<100 を列挙 | 乗算ブラック(K90+)+オーバープリント併用=警告 / その他=情報 |
| グラデーションメッシュ | MeshItem を列挙 | 警告(分版トラブル要因) |
| パターン使用 | PatternColor の塗り/線を列挙 | 情報 |
| オブジェクト総数 | 走査オブジェクト総数が閾値以上 | 警告(RIP負荷) / 未満=情報(件数表示) |
| 細ケイ+薄アミ | 線幅が閾値(0.5pt)以下かつインキ総量が閾値(20%)以下の線 | 警告(飛び・カスレ要因)。ヘアラインチェックとは別判定 |

### その他

| 項目 | ロジック | 判定 |
|---|---|---|
| オーバープリント | fillOverprint / strokeOverprint = true を列挙 | 白(CMYK全0)にノセ=エラー(白ノセ事故) / その他=警告(意図的な場合あり) |
| ヘアライン/極細線 | strokeWidth = 0(ヘアライン)、0超〜最小線幅以下 | エラー |
| ラスタライズ効果解像度 | document.rasterEffectSettings.resolution | 下限未満=エラー(現在値表示) |
| 不要オブジェクト | 非表示、非表示レイヤー内、ロック済み非表示、空テキストフレーム、孤立点(1点パス)、アートボード+塗り足し範囲外 | 警告(意図的な場合があるため) |

各項目には「原因と対応」の短い解説が付き、結果画面の詳細欄・HTMLレポート・CSVに表示されます(教育用途)。

## 3. 結果画面とレポート

- 最上部に総合判定(「✔ 全項目OK」/「✖ エラー n件・警告 m件・情報 k件」)と、検出した仕上がりサイズ(**左右×天地 mm**)を表示します。
- 左のツリーでカテゴリ > 項目(件数付き)を選択すると、右に検出オブジェクト一覧が出ます。
- 行を選択して **「選択してズーム」** ボタン、または **行のダブルクリック** で、該当オブジェクトを選択状態にしてビューをズームします(目視確認用)。ズーム倍率は 50〜400% にクランプされます。
- **ロック/非表示オブジェクトの選択**: Illustrator はロック済み・非表示のオブジェクト(ロック/非表示レイヤー・グループ内のものを含む)を選択できないため、選択実行時に該当のロック/非表示を**自動的に解除してから**選択します。解除した内容(「レイヤー『◯◯』のロックを解除しました」等)はリスト下のステータス欄に表示されます。**ロックは元に戻しません**(戻すと選択が見えなくなるため)。検版後に必要に応じて手動でロックし直してください。解除しても選択が反映されない場合(ガイド等)はその理由をステータス欄に表示し、位置までズームだけ行います。
- **実行中の中断**: 検版実行中は **ESC キーを押し続けると中断**できます(「中断」ボタンも設置していますが、ExtendScript の制約によりスクリプト実行中はボタンが反応しないことがあります。確実なのは ESC キーです)。中断時は途中結果を破棄して設定画面に戻ります。
- **ウィンドウ操作**: ダイアログは端をドラッグしてリサイズできます。設定画面はウィンドウを縮めるとスクロールバーが出現し、下部のボタン列は常に表示されます。設定画面では ESC=キャンセル(何もせず終了・設定も保存しない)/ Enter=検版実行 です。
- **レポート保存(HTML/CSV)**: 保存先フォルダを指定すると `<ドキュメント名>_kenpan_<日時>.html / .csv`(UTF-8 BOM付き)を書き出します。ファイル名・日時・設定値・項目別判定・検出一覧・解説を含みます。

## 4. 制限事項

- **ES3 制約**: ExtendScript の言語仕様上、JSON・ES5以降の配列メソッド等は使用していません。設定はテキスト(key=value)保存です。
- **埋め込み画像(RasterItem)の実効解像度**: ExtendScript から元ピクセル数を直接取得できないため **判定不能=警告** とします(誤判定を避けるため)。リンク画像に切り替えるか、Photoshop 側で確認してください。
- **リンク画像の判定範囲**: ファイルヘッダ自前解析は JPEG / PNG / TIFF / PSD / BMP / GIF のみ対応。EPS / AI / PDF / WebP 等は判定不能=警告になります。
- **ヘッダ解析は先頭64KBのみ**: 処理速度確保(フリーズ防止)のため、画像ファイルは先頭64KBだけを読んで解析します。TIFF の IFD や JPEG の SOF マーカーがこの範囲より後ろにある特殊なファイルは判定不能=警告になります(誤判定はしません)。同一ファイルの解析結果はカラーモード/解像度チェック間でキャッシュ共有され、二重読みしません。
- **TIFF の 2値判定**: BitsPerSample が複数サンプルの場合はオフセット参照になるため、判定できないケースがあります(その場合カラー/グレー扱い)。
- **特色のインキ総量**: 代替カラー定義(spot.color)×濃度からの近似値です。実際のインキとは異なる場合があります。
- **画像内部の色・インキ総量**: 走査対象外です(注記の上、要確認扱い)。RIP・Acrobat 側で確認してください。
- **パターン・シンボル・グラフの内部**: 内部オブジェクトの色・線幅は走査できません。
- **アピアランス効果由来の色・透明**: 効果(ドロップシャドウ等)が生成する色・透明は検出できません。透明の分割・統合プレビューを併用してください。
- **トンボ検出はヒューリスティック**: 細線群の位置から推定するため、独自形状のトンボは検出できないことがあります(警告=目視確認)。
- **文字セーフマージン/塗り足し判定** は矩形バウンディングボックスに基づくため、回転・変形オブジェクトでは実際より厳しめに出ることがあります。

## 5. 対象外項目(Acrobat「プリフライト」プロファイル相当のうち実装しないもの)

以下は **.ai ドキュメント上に存在しない、または PDF 書き出し後にはじめて生じる概念** のため本ツールでは対象外です。
**PDF 書き出し後に Acrobat のプリフライト機能で別途確認してください。**

| 項目 | 対象外の理由 |
|---|---|
| PDF の暗号化/破損/構文エラー | PDFファイル構造の検査。.ai ドキュメントには存在しない |
| PDF バージョン / PDF/X 準拠 | PDF 書き出し設定で決まる。書き出し前にはチェック不能 |
| 出力インテント / ICC プロファイルの埋め込み | PDF/X の出力インテントは書き出し時に付与される概念 |
| .notdef グリフ / Unicode マッピング / フォント埋め込み | フォント埋め込みは PDF 固有。Illustrator 上ではアウトライン化チェックで代替 |
| 非可逆圧縮(JPEG圧縮率)/ ダウンサンプルによる劣化 | PDF 書き出し設定(圧縮・ダウンサンプル)の領分 |
| Acrobat レイヤー(OCG) | PDF のレイヤー構造。書き出し設定で決まる |
| 面付け用トンボの形状・位置検証 | 面付けは後工程(面付けソフト/RIP)の領分。本ツールは日本式トンボの有無推定まで |
| Normalized PDF(印刷用正規化PDF)の二重チェック | 正規化は PDF ワークフロー側の処理 |

## 6. チェック項目の追加方法(開発者向け)

チェックはレジストリ方式です。関数を1個追加するだけで項目を増やせます。

1. `defaultConfig()` の `checks` に `新ID: true` を追加。
2. `getCheckIdOrder()` の配列の表示したい位置に `"新ID"` を追加。
3. `CHECK_META` に `新ID: { category: "カテゴリ名", name: "表示名", advice: "原因と対応の解説" }` を追加。(カテゴリを新設する場合は `CATEGORY_ORDER` にも追加)
4. チェック関数を登録:

```javascript
CHECKS.新ID = function (doc, cfg, ctx) {
    // ctx.items … 全オブジェクトの走査結果(1回の再帰走査を全チェックで共有)
    //   各要素: { item, typename, layerName, hiddenSelf, hiddenInherited,
    //             lockedSelf, lockedInherited, geometricBounds, visibleBounds }
    // ctx.artboardTrims … アートボードごとの仕上がり枠検出結果
    var details = [];
    var ngCount = 0;
    for (var i = 0; i < ctx.items.length; i++) {
        var rec = ctx.items[i];
        // 判定して details.push(detailItem("説明文", rec.item));
    }
    var status = ngCount > 0 ? "NG" : "OK"; // "NG"|"WARN"|"INFO"|"OK"|"SKIP"
    return makeResult("新ID", status, "概要文", details, "備考(任意)");
};
```

ドキュメント全体の再帰走査は `scanDocument()` が1回だけ行い、全チェックが `ctx.items` を共有します。
チェック関数内で `doc.pageItems` を直接回さないでください(大規模データで遅くなります)。

## 7. 動作確認手順(Illustrator 上で)

1. Illustrator で任意の入稿データ(または新規ドキュメント)を開く。
2. [ファイル] > [スクリプト] から本スクリプトを実行し、設定ダイアログが日本語で表示されることを確認。
3. 仕上がりサイズ・カラー数を実データに合わせて設定し「検版実行」。
4. プログレスバー進行後、総合判定・仕上がりサイズ・ツリーが表示されることを確認。
5. 動作テスト用の仕込み例:
   - テキストを1つ残す → 「アウトライン化」がエラーになる
   - RGB の矩形を置く → 「RGB混入」がエラーになる
   - 0.1pt の線を引く → 「ヘアライン/極細線」がエラーになる
   - C40M40Y40K100 の小さい文字 → 「リッチブラック/4Cブラック」がエラーになる
   - 何も置いていないアートボードを追加 → 「白ページ」が警告になる
6. 検出一覧の行をダブルクリックし、該当オブジェクトが選択・ズームされることを確認。
7. 「レポート保存(HTML)」で保存し、ブラウザで日本語が正しく表示されることを確認。
8. 「設定を保存」→ Illustrator を再起動 → 再実行し、設定が復元されることを確認。
